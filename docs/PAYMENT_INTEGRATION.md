# 微信 / 支付宝 支付接入说明

当前接口 `POST /api/bookings/{id}/pay` 在用户选择渠道后直接标记为「已付款」，便于演示。**正式上线前**应改为：先调起真实支付，**支付平台异步通知（回调）成功后再**更新订单状态。

## 微信支付（JSAPI / 小程序 / APP）

你需要向开发同学提供：

1. **商户号** `mchid`
2. **应用 AppID**（公众号/小程序/开放平台应用）
3. **API v3 密钥**（32 位）
4. **商户 API 证书**：`apiclient_cert.pem`、`apiclient_key.pem`（或 P12）
5. **支付结果通知 URL**（需公网 HTTPS，由后端配置）

后端可引入官方 SDK：`com.github.wechatpay-apiv3:wechatpay-java`（或按文档自行验签、调下单接口）。

## 支付宝（手机网站 / APP）

你需要提供：

1. **应用 AppID**
2. **应用私钥**（RSA2）
3. **支付宝公钥**（或证书模式下的证书）
4. **异步通知 notify_url**（公网 HTTPS）

后端可引入：`com.alipay.sdk:alipay-sdk-java`。

## 闲鱼

闲鱼一般为**线下/平台内沟通转账**，无开放支付 API。当前实现：用户选择「闲鱼」并确认后，与微信/支付宝演示流程一样先记为已付款；你可改为「待管理员核对到账后再改已付款」。

## 建议流程

1. 用户点「付款」→ 后端创建**预支付单**（金额、单号）→ 返回调起支付所需参数给前端。
2. 用户完成支付 → 微信/支付宝 **POST 回调** 你的服务端 → 验签通过后 `UPDATE booking SET status=2, payment_channel=...`。
3. 前端可轮询订单状态或支付页返回后查询订单。

将以上材料（可脱敏的配置项名称 + 测试/生产环境说明）交给后端即可对接。
